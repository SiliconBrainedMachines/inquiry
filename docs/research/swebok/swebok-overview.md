# SWEBOK Overview and Structure

## What It Is

The **Software Engineering Body of Knowledge (SWEBOK)** is an IEEE standard (ISO/IEC TR 19759) that defines the scope of software engineering as a discipline. It identifies what constitutes generally accepted knowledge in the field.

SWEBOK does not prescribe methods — it describes what a competent software engineer should know.

## The 15 Knowledge Areas (v3, 2014)

| # | Knowledge Area | Scope |
|---|---------------|-------|
| 1 | Software Requirements | Elicitation, analysis, specification, validation |
| 2 | Software Design | Architecture, detailed design, design patterns |
| 3 | Software Construction | Coding, unit testing, debugging, integration |
| 4 | Software Testing | Test levels, techniques, test-related measures |
| 5 | Software Maintenance | Correction, adaptation, perfection, prevention |
| 6 | Software Configuration Management | Identification, control, status accounting, auditing |
| 7 | Software Engineering Management | Planning, enactment, review, measurement |
| 8 | Software Engineering Process | Definition, implementation, assessment, improvement |
| 9 | Software Engineering Models and Methods | Modeling, formal methods, types of models |
| 10 | Software Quality | Quality assurance, verification, validation |
| 11 | Software Engineering Professional Practice | Ethics, professionalism, group dynamics |
| 12 | Software Engineering Economics | Cost estimation, ROI, decision analysis |
| 13 | Computing Foundations | Problem solving, abstraction, algorithms |
| 14 | Mathematical Foundations | Logic, sets, graphs, probability |
| 15 | Engineering Foundations | Empirical methods, statistical analysis, measurement |

## v4 Additions (2024)

SWEBOK v4 added coverage for:

- **AI/ML in Software Engineering** — automated code generation, AI-assisted testing
- **Architecture** — elevated from a subtopic of Design to its own emphasis
- **DevOps and Continuous Practices** — CI/CD as an engineering discipline
- **Security** — integrated across KAs rather than isolated

## Structure of Each KA

Every Knowledge Area follows a consistent structure:

1. **Breakdown of topics** — hierarchical decomposition
2. **Matrix of topics vs. reference material** — maps topics to canonical sources
3. **Recommended references** — books and standards for each topic

## Key Property

SWEBOK is descriptive, not prescriptive. It says "this knowledge exists and matters" but does not mandate how to apply it. The application is left to processes, methods, and tools chosen by the practitioner.
