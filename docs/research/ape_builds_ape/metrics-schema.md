# Metrics Schema

Schema for `.ape/metrics.yaml` (primary) → `docs/issues/NNN-slug/metrics.yaml` (conditional copy) — structured experiment data per APE cycle.

---

## Schema

```yaml
# .ape/metrics.yaml (primary) → docs/issues/NNN-slug/metrics.yaml (conditional copy)
# One file per APE cycle (issue). All fields optional except issue.

issue: 68                          # GitHub issue number (required)
version: "0.0.14"                  # Version produced by this cycle (if any)
model: "claude-sonnet-4"        # Primary model used during cycle
agent: "copilot"                   # Agent runtime (copilot | crush | local)

cycle:
  completed: true                  # Did the cycle reach IDLE again?
  aborted: false                   # Was the cycle abandoned?
  darwin_activated: true           # Was EVOLUTION triggered?
  darwin_issue: 70                 # Issue # generated by DARWIN (if any)

timing:
  branch_created: "2026-04-19T10:00:00Z"   # ISO 8601
  pr_merged: "2026-04-19T12:30:00Z"        # ISO 8601
  # cycle_hours is derived, not stored

plan:
  total_phases: 5                  # Number of phases in plan.md
  completed_phases: 5             # Phases with [x] at merge time
  deviations: 0                    # Phases added/removed/reordered vs original

tests:
  before: 127                      # Test count before this cycle
  after: 131                       # Test count after this cycle
  delta: 4                         # Net new tests

delta_failures:                    # Times LLM didn't follow δ (prompt)
  count: 0                         # Total retries/corrections needed
  notes: ""                        # Brief description of what failed

observations: |                    # Freeform notes (optional)
  Anything notable about this cycle.
  Transferred from mutations.md if relevant.
```

## Field Reference

| Field | Type | Required | Description |
|---|---|---|---|
| `issue` | int | **yes** | GitHub issue number |
| `version` | string | no | Version tag if this cycle produced a release |
| `model` | string | no | LLM model identifier |
| `agent` | string | no | Agent runtime: `copilot`, `crush`, `local`, `other` |
| `cycle.completed` | bool | no | Cycle reached IDLE (default: true if PR merged) |
| `cycle.aborted` | bool | no | Cycle was abandoned (branch deleted without merge) |
| `cycle.darwin_activated` | bool | no | EVOLUTION state was entered |
| `cycle.darwin_issue` | int | no | Issue number generated by DARWIN |
| `timing.branch_created` | datetime | no | When feature branch was created. Captured via snapshot at IDLE → ANALYZE transition (`.ape/metrics_snapshot.yaml`). |
| `timing.pr_merged` | datetime | no | When PR was merged |
| `plan.total_phases` | int | no | Phases in plan.md |
| `plan.completed_phases` | int | no | Phases marked [x] at merge |
| `plan.deviations` | int | no | Count of plan changes vs original |
| `tests.before` | int | no | Test count at cycle start. Captured via snapshot at IDLE → ANALYZE transition (`.ape/metrics_snapshot.yaml`). May be approximate. |
| `tests.after` | int | no | Test count at cycle end |
| `tests.delta` | int | no | Net change |
| `delta_failures.count` | int | no | Times δ failed (prompt non-compliance) |
| `delta_failures.notes` | string | no | What went wrong |
| `observations` | string | no | Freeform notes |

## Example: Issue #68

```yaml
issue: 68
version: "0.0.14"

agent: "copilot"

cycle:
  completed: true

  darwin_activated: false

  total_phases: 5

  deviations: 0

tests:
  before: 127
  after: 131
  delta: 4

delta_failures:
  count: 0
  notes: ""

observations: |
  Clean cycle. No deviations. EVOLUTION infrastructure added without issues.
```

## Notes

- **Primary storage:** `.ape/metrics.yaml` — generated by DARWIN during EVOLUTION.
- **Conditional copy:** copied to `docs/issues/NNN-slug/metrics.yaml` only if `evolution.enabled: true` AND `remote origin ≈ ccisnedev/finite_ape_machine`.
- **Two-point capture:** `tests.before` and `timing.branch_created` are captured at cycle start (IDLE → ANALYZE) in `.ape/metrics_snapshot.yaml`. All other fields are collected at cycle end (EVOLUTION).
- **Complete cycles only:** `metrics.yaml` is only generated for cycles that reach EVOLUTION. Partial or aborted cycles do not produce metrics.
- **Maturity roadmap (D7):** Stage 1 = orchestrator prompt (current). Stage 2 = sub-agent. Stage 3 = `ape metrics collect` CLI command.
- Future: `ape` CLI command to auto-generate from git/GitHub data.
- Fields with `no` required can be omitted — extraction scripts handle missing fields gracefully.
- `timing.pr_merged` may be empty at generation time (PR not yet merged); can be filled post-merge.
